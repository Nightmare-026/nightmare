Database Schema — Supabase PostgreSQL
CRITICAL CHANGES IN V3.0:
NEW: subjects table for subject folder system
UPDATED: products table now has subject_id foreign key
UPDATED: users table — admin role removed, access controlled by DEVELOPER_EMAIL only
FIXED: categories — only 4 categories, pre-seeded, cannot be edited

Table: users
id (UUID PK), email (unique), password_hash, name, avatar_url, is_banned (bool, default false), created_at, updated_at
NOTE: role field REMOVED. Admin access is determined by checking if user.email === process.env.DEVELOPER_EMAIL
Table: categories (FIXED)
id (UUID PK), name, slug (unique), icon (emoji string), description, display_order (int), is_active (bool), created_at
FIXED CATEGORIES (pre-seeded in migration, cannot be changed):
1. Class 1-8 (slug: class-1-8)
2. Class 9-10 (slug: class-9-10)
3. Polytechnic (CSE) (slug: polytechnic-cse)
4. Polytechnic (EE) (slug: polytechnic-ee)
Table: subjects (NEW)
id (UUID PK), name (e.g., Physics, Data Structures), slug (unique), category_id (FK to categories), description (text, nullable), display_order (int), is_active (bool, default true), created_at, updated_at
Admin can create unlimited subjects within each category from the admin panel.
Table: products (UPDATED)
id (UUID PK), title, slug (unique), description (text), category_id (FK), subject_id (FK to subjects, nullable for wallpapers), type (ENUM: pdf/wallpaper), price (decimal, nullable), is_free (bool), file_url (S3 for pdf / Cloudinary for wallpaper), preview_url (watermarked), thumbnail_url (400x300), tags (string[]), downloads_count (int, default 0), is_active (bool, default true), created_at, updated_at
CHANGE: Added subject_id FK. For PDFs, this links to the subject folder. For wallpapers, this can be null.
Table: orders
id (UUID PK), user_id (FK users), product_id (FK products), amount (decimal), status (ENUM: pending/paid/failed/refunded), razorpay_order_id (unique), razorpay_payment_id (nullable, unique), razorpay_signature (nullable), created_at, updated_at
Table: download_tokens
id (UUID PK), user_id (FK users), product_id (FK products), token (UUID unique), order_id (FK orders, nullable — null for free downloads), expires_at (timestamp — 24 hours from creation), download_count (int, default 0), max_downloads (int, default 3), created_at
Table: wishlists
id (UUID PK), user_id (FK users), product_id (FK products), created_at. Unique constraint on (user_id, product_id)
Table: product_reviews
id (UUID PK), product_id (FK products), user_id (FK users), rating (int 1-5), review_text (text, nullable), is_verified_purchase (bool — true only if user has paid order for product), is_approved (bool, default true), helpful_count (int, default 0), photo_url (nullable), created_at
Table: quick_reactions
id (UUID PK), product_id (FK products), user_id (UUID nullable — allow anonymous), reaction_type (ENUM: fire/premium/creative/useful), ip_address, created_at. Unique constraint on (product_id, user_id) OR (product_id, ip_address) for anonymous
Table: site_feedback
id (UUID PK), section_name (e.g. hero/products/footer), page_url, rating (int 1-5), message (text, nullable), email (nullable), user_id (UUID nullable), created_at. On INSERT: trigger email to FEEDBACK_EMAIL
Table: promo_codes
id (UUID PK), code (unique uppercase), discount_type (ENUM: percent/flat), discount_value (decimal), usage_limit (int nullable), usage_count (int default 0), min_order_amount (decimal nullable), expires_at (nullable), is_active (bool default true), created_at
Table: email_subscribers
id (UUID PK), email (unique), is_active (bool default true), source (ENUM: footer/popup/checkout), created_at

Admin Panel (UPDATED - Developer Only Access)
CRITICAL SECURITY UPDATE:
Users table NO LONGER has a 'role' field
Admin access is determined ONLY by comparing user.email with DEVELOPER_EMAIL env variable
If email doesn't match: return 403 immediately, do not process request
Frontend should hide admin links for non-developer users entirely
No user can become admin through database manipulation

Backend Middleware: developerOnly.ts
```typescript
// backend/src/middleware/developerOnly.ts
import { Request, Response, NextFunction } from 'express';

export const developerOnly = (req: Request, res: Response, next: NextFunction) => {
  const user = req.user; // Set by auth middleware
  
  if (!user) {
    return res.status(401).json({ success: false, error: 'UNAUTHORIZED' });
  }
  
  if (user.email !== process.env.DEVELOPER_EMAIL) {
    return res.status(403).json({ 
      success: false, 
      error: 'ACCESS_DENIED',
      message: 'Admin access is restricted to authorized developer only'
    });
  }
  
  next();
};
```
